# 欢雨鲜花店项目 - 命令解释文档

## 1. 项目概述

欢雨鲜花店是一个基于前后端分离架构的电商平台，主要功能包括商品管理、用户管理、订单管理、购物车管理、评价管理等。

**技术栈**：
- 前端：Vue 3 + Vite + Pinia + Vue Router + Axios + Element Plus + Tailwind CSS
- 后端：ASP.NET Core 9.0 + Entity Framework Core + MySQL + JWT

## 2. 前端命令

| 命令 | 描述 | 用途 |

| `npm install` | 安装前端项目依赖 | 首次克隆项目或依赖更新后使用 |
| `npm run dev` | 启动前端开发服务器 | 开发过程中使用，支持热更新 |

### 2.2 命令详解

- 首次克隆项目后

#### 2.2.3 `npm run build`

**功能**：构建生产版本

- 输出目录：`dist/`

## 3. 后端命令

### 3.1 基础命令

| 命令 | 描述 | 用途 |
|------|------|------|
| `dotnet restore` | 恢复项目依赖 | 首次克隆项目或依赖更新后使用 |
| `dotnet build` | 构建后端项目 | 编译项目，检查语法错误 |
| `dotnet run` | 运行后端项目 | 启动后端开发服务器 |
| `dotnet test` | 运行后端测试 | 执行单元测试和集成测试 |
| `dotnet publish` | 发布后端项目 | 构建生产部署包 |
| `dotnet ef migrations add <MigrationName>` | 添加数据库迁移 | 当数据库模型发生变化时 |
| `dotnet ef database update` | 更新数据库 | 应用迁移，更新数据库结构 |

### 3.2 命令详解

#### 3.2.1 `dotnet restore`

**功能**：恢复项目依赖
**使用场景**：
- 首次克隆项目后
- .csproj 文件更新后
- 依赖版本发生变化时

**示例**：
```bash
cd backend/HuanyuFlowerShop
dotnet restore
```

#### 3.2.2 `dotnet build`

**功能**：构建后端项目
**配置**：
- 目标框架：net9.0
- 输出目录：`bin/Debug/net9.0/` 或 `bin/Release/net9.0/`

**示例**：
```bash
cd backend/HuanyuFlowerShop
dotnet build
```

#### 3.2.3 `dotnet run`

**功能**：启动后端开发服务器
**配置**：
- 默认端口：5002
- 监听地址：http://0.0.0.0:5002 和 http://localhost:5002
- 自动加载环境配置

**示例**：
```bash
cd backend/HuanyuFlowerShop
dotnet run
```

**输出**：
```

      Now listening on: http://0.0.0.0:5002

      Now listening on: http://localhost:5002
```

#### 3.2.4 `dotnet ef migrations add <MigrationName>`

**功能**：添加数据库迁移
**前提**：
- 已安装 Entity Framework Core Tools
- 数据库连接字符串已配置

**示例**：
```bash

dotnet ef migrations add AddProductReviewTable
```

#### 3.2.5 `dotnet ef database update`

**功能**：更新数据库
**用途**：应用所有未应用的迁移，更新数据库结构

**示例**：
```bash
cd backend/HuanyuFlowerShop
dotnet ef database update
```

## 7. 常见问题和解决方案

### 7.1 前端问题

#### 7.1.1 依赖安装失败

**问题**：`npm install` 安装依赖失败
**解决方案**：
- 检查网络连接
- 清除 npm 缓存：`npm cache clean --force`
- 使用国内镜像：`npm config set registry https://registry.npmmirror.com`
- 删除 node_modules 目录后重新安装

#### 7.1.2 开发服务器启动失败

**问题**：`npm run dev` 启动失败
**解决方案**：
- 检查端口是否被占用
- 检查 vite.config.js 配置
- 查看错误日志，根据提示修复问题

### 7.2 后端问题

#### 7.2.1 数据库连接失败

**问题**：`dotnet run` 或数据库迁移时连接失败
**解决方案**：
- 检查 appsettings.json 中的连接字符串
- 确保 MySQL 服务正在运行
- 检查数据库用户权限

#### 7.2.2 迁移失败

**问题**：`dotnet ef migrations add` 或 `dotnet ef database update` 失败
**解决方案**：
- 检查模型定义是否正确
- 确保数据库连接正常
- 查看错误日志，根据提示修复问题
