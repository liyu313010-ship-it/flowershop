## 项目概述

**欢雨花艺电商系统**是一个基于现代Web技术栈开发的电子商务平台，专注于花卉产品的展示、销售和配送服务。

系统主要功能包括：
- 商品浏览与搜索
- 用户注册与登录
- 购物车管理
- 订单处理
- 个人中心
- 地址管理
- 支付功能

## 技术栈概览

本项目采用了当前流行的前后端分离架构，使用不同的技术栈来实现前端展示和后端业务逻辑。

### 前端技术栈
- **Vue 3** - 渐进式JavaScript框架，用于构建用户界面
- **Vite** - 现代前端构建工具，提供极速的开发体验
- **Pinia** - Vue的状态管理库，管理应用程序的状态
- **Vue Router** - Vue官方的路由管理器，实现页面间的导航
- **Tailwind CSS** - 实用优先的CSS框架，用于快速构建UI
- **Axios** - 用于发起HTTP请求的库，与后端API交互

### 后端技术栈
- **ASP.NET Core** - 跨平台的高性能开源框架，用于构建现代化云应用
- **C#** - 面向对象的编程语言，用于开发后端逻辑
- **Entity Framework Core (EF Core)** - 微软的ORM框架，用于数据库操作
- **JWT (JSON Web Token)** - 用于用户认证和授权
- **MySQL** - 成熟稳定的关系型数据库，用于生产环境

## 前端技术详解

### Vue 3

Vue 3是当前项目的前端核心框架，它使用了**Composition API**，这种API设计更加灵活，可以更好地组织和重用代码。

```javascript
// Vue 3 Composition API 示例
import { ref, computed, onMounted } from 'vue'

export default {
  setup() {
    const count = ref(0)
    const doubleCount = computed(() => count.value * 2)
    
    function increment() {
      count.value++
    }
    
    onMounted(() => {
      console.log('组件已挂载')
    })
    
    return {
      count,
      doubleCount,
      increment
    }
  }
}
```

### Vite

Vite是一个现代化的前端构建工具，它为Vue 3项目提供了极快的开发服务器启动时间和热模块替换（HMR）能力。相比传统的Webpack，Vite在开发阶段能够提供更好的开发体验。

### Pinia

Pinia是Vue官方推荐的状态管理库，它取代了Vue 2中的Vuex。Pinia提供了更简洁的API和更好的TypeScript支持。

```javascript
// Pinia Store 示例
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0
  }),
  actions: {
    increment() {
      this.count++
    }
  },
  getters: {
    doubleCount: (state) => state.count * 2
  }
})
```

### Vue Router

Vue Router用于实现SPA（单页应用）的路由功能，它允许用户在不重新加载页面的情况下浏览不同的视图。

### Tailwind CSS

Tailwind CSS是一个实用优先的CSS框架，它提供了大量的原子类，开发者可以直接在HTML中使用这些类来设计UI，而无需编写大量的自定义CSS。

```html
<!-- Tailwind CSS 示例 -->
<button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
  点击我
</button>
```

## 后端技术详解

### ASP.NET Core

ASP.NET Core是微软推出的跨平台Web应用框架，它可以在Windows、macOS和Linux上运行。ASP.NET Core提供了高性能的HTTP请求处理能力，并支持依赖注入等现代开发模式。

### C#

C#是微软开发的现代面向对象编程语言，它是ASP.NET Core的主要开发语言。

```csharp
// C# 控制器示例
[ApiController]
[Route("api/[controller]")]
public class ProductController : ControllerBase
{
    [HttpGet]
    public IActionResult GetProducts()
    {
        // 获取产品数据的逻辑
        return Ok(products);
    }
}
```

### Entity Framework Core

Entity Framework Core是一个ORM（对象关系映射）框架，它允许开发者使用面向对象的方式操作数据库，而无需编写原始的SQL查询。

```csharp
// EF Core 数据库上下文示例

{
    public DbSet<Product> Products { get; set; }
    public DbSet<Order> Orders { get; set; }
    public DbSet<User> Users { get; set; }
    
    public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options) : base(options)
    {}
    
    // 注意：在ASP.NET Core中，数据库连接通常在Program.cs中配置，而不是在这里
}
```

### JWT认证

JWT (JSON Web Token) 是一种用于在网络应用间安全传输声明的开放标准。在本项目中，JWT用于用户认证，确保只有授权用户才能访问受保护的API。

## 前后端交互

前后端通过RESTful API进行交互，前端使用Axios发送HTTP请求到后端API。

### 前端API调用示例

```javascript
// Axios API调用示例
import axios from 'axios'

// 创建axios实例
const api = axios.create({
  baseURL: 'http://localhost:5232/api',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 请求拦截器，添加JWT token
api.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  error => {
    return Promise.reject(error)
  }
)

export default api
```

  }

## 开发环境搭建

### 前端开发环境

1. **安装Node.js**：前往[Node.js官网](https://nodejs.org/)下载并安装最新版本
2. **安装依赖**：在前端目录下运行`npm install`
3. **启动开发服务器**：运行`npm run dev`
4. **构建生产版本**：运行`npm run build`

### 后端开发环境

1. **安装.NET SDK**：前往[微软官网](https://dotnet.microsoft.com/download)下载并安装最新版本
2. **安装依赖**：在后端目录下运行`dotnet restore`
3. **启动开发服务器**：运行`dotnet run`
4. **构建项目**：运行`dotnet build`

## 常见问题

### 1. MySQL连接字符串配置

在ASP.NET Core中，MySQL的连接字符串通常配置在appsettings.json文件中：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "server=localhost;port=3306;database=huanyu_flower_shop;user=root;password=your_password;"
  },
  "Logging": {
    // 其他配置...
  }
}
```

### 2. 前后端跨域问题

在开发过程中，前端和后端可能运行在不同的端口，这会导致跨域问题。解决方法是在后端配置CORS（跨域资源共享）。

```csharp
// 在Program.cs中配置MySQL数据库连接
builder.Services.AddDbContext<ApplicationDbContext>(options =>
    options.UseMySql(
        builder.Configuration.GetConnectionString("DefaultConnection"),
        ServerVersion.AutoDetect(builder.Configuration.GetConnectionString("DefaultConnection"))
    ));

// 在Program.cs中配置CORS
builder.Services.AddCors(options =>
{
    options.AddDefaultPolicy(builder =>
    {
        builder.WithOrigins("http://localhost:5175")
               .AllowAnyHeader()
               .AllowAnyMethod()
               .AllowCredentials();
    });
});

// 使用CORS中间件
app.UseCors();
```

### 2. JWT认证失败

如果遇到JWT认证失败，需要检查以下几点：
- Token是否过期
- Token是否正确
- 后端JWT配置是否正确

### 3. 数据库连接问题

如果无法连接数据库，需要检查：
- 数据库文件是否存在
- 数据库连接字符串是否正确
- 数据库权限设置是否正确
